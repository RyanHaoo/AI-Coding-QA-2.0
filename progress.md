# 建筑施工质检情报员开发进度

**最后更新**：2026-07-03  
**进度口径**：使用 AI Agent Coding 的开发节奏估算，不等同于真实生产排期。  
**当前决策**：Supabase Auth 登录、Vercel 部署、MVP 演示级权限控制；Coze 知识接口在对应阶段进入时再细化。

## 进度追踪规则

- 本文件是项目阶段进度的单一追踪入口。
- 每个阶段必须作为一个独立 speckit SDD 单元推进，包含清晰的 spec、plan、tasks 和验收标准。
- 每个阶段开发完成后，必须回到本文件更新阶段状态、完成日期、主要交付、验证结果、遗留问题和下一阶段入口条件。
- 每个阶段必须遵循“先框架再模块”“先核心再拓展”：先保证可登录、可看数据、可完成核心工单闭环，再扩展助手、管理端、知识问答和上线能力。
- 估算中的“段”表示一段完整 AI Coding 对话，通常覆盖 speckit 文档、实现、静态检查和验收说明。

# 契约: 工单详情操作流程

## 范围

本契约描述阶段 3 的页面状态、操作入口、表单输入、服务端写入结果和可观察行为。它不是公开 REST API 规范；阶段 3 不新增独立 API route。

## 页面入口

| 入口 | 当前角色 | 页面意图 |
|------|----------|----------|
| `/?view=tickets&ticketId={id}` | 质检员、施工方 | 从普通工单列表打开详情并按权限操作 |
| `/?view=admin-tickets&ticketId={id}` | 管理员 | 从管理员工单中心打开详情并按状态规则操作 |

## 操作入口可见性

| 操作 | 可见角色/关系 | 前置状态 |
|------|---------------|----------|
| 编辑 | 发起人、当前责任人、管理员 | 待处理 |
| 解决 | 当前责任人、管理员 | 待处理 |
| 拒绝 | 当前责任人、管理员 | 待处理 |
| 指派他人 | 当前责任人、管理员 | 待处理 |
| 重新打开 | 发起人、管理员 | 已完成或已拒绝 |

无权操作不展示入口。服务端提交时必须重新校验同一规则；即使用户构造提交，也必须被拒绝。

## 表单契约

### 编辑工单

**输入字段**

- `ticketId`: 必填，工单 id
- `severity`: 必填，轻微、一般、严重、紧急之一
- `specialty`: 必填，建筑设计专业、结构专业、给排水专业之一
- `summary`: 必填，问题描述
- `locationDetail`: 必填，详细位置
- `description`: 选填，问题详情
- `existingImageUrls`: 选填，保留的原现场图片 URL
- `newImages`: 选填，新增图片文件或等效上传结果

**成功结果**

- 工单仍为待处理。
- 只更新允许编辑的问题信息字段。
- 新增一条“编辑”处理记录。
- 页面返回展示状态，详情展示最新问题信息和现场图片。

**失败结果**

- 缺少必填字段时，表单保留输入并提示补全。
- 工单不再是待处理或当前身份无权编辑时，拒绝提交并提示刷新或返回详情查看最新状态。

### 解决工单

**输入字段**

- `ticketId`: 必填，工单 id
- `rootCause`: 必填，问题归因
- `preventiveAction`: 选填，预防建议

**成功结果**

- 工单状态变为已完成。
- `root_cause` 写入问题归因。
- `preventive_action` 写入预防建议或保持为空。
- 新增一条“解决”处理记录。
- 列表中该工单进入已结束范围。

**失败结果**

- 缺少问题归因时，阻止提交并提示补全。
- 当前身份不是当前责任人或管理员时，拒绝提交。
- 工单不再是待处理时，拒绝提交。

### 拒绝工单

**输入字段**

- `ticketId`: 必填，工单 id
- `reason`: 必填，拒绝原因

**成功结果**

- 工单状态变为已拒绝。
- 新增一条“拒绝”处理记录，记录拒绝原因。
- 列表中该工单进入已结束范围。

**失败结果**

- 缺少拒绝原因时，阻止提交并提示补全。
- 当前身份不是当前责任人或管理员时，拒绝提交。
- 工单不再是待处理时，拒绝提交。

### 指派他人

**输入字段**

- `ticketId`: 必填，工单 id
- `assigneeMembershipId`: 必填，当前项目内另一名施工方项目身份
- `reason`: 必填，指派原因

**成功结果**

- 工单状态保持待处理。
- 当前责任人变为所选施工方身份。
- 新增一条“指派他人”处理记录，记录指派原因和责任人变化。
- 原责任人刷新列表后不再看到该工单，新责任人刷新列表后可以看到该工单。

**失败结果**

- 缺少新责任人或指派原因时，阻止提交并提示补全。
- 新责任人不是当前项目施工方或等于原责任人时，拒绝提交。
- 当前身份不是当前责任人或管理员时，拒绝提交。

### 重新打开

**输入字段**

- `ticketId`: 必填，工单 id
- `reason`: 必填，重新打开原因

**成功结果**

- 工单状态恢复为待处理。
- 当前责任人保持不变。
- 原拒绝原因、问题归因、预防建议和历史处理记录继续可见。
- 新增一条“重新打开”处理记录。
- 工单重新进入责任人的待处理列表。

**失败结果**

- 缺少重新打开原因时，阻止提交并提示补全。
- 当前身份不是发起人或管理员时，拒绝提交。
- 工单不是已完成或已拒绝时，拒绝提交。

## 指派候选人契约

**Given** 用户打开待处理工单详情且有权指派  
**When** 系统展示指派表单  
**Then** 候选人只包含当前项目内角色为施工方、且不是当前责任人的项目身份。

若没有可选施工方，指派入口可以隐藏或展示禁用状态，并说明当前项目暂无其他施工方可指派。

## 图片编辑契约

**Given** 用户有权编辑待处理工单  
**When** 用户进入编辑模式  
**Then** 现场图片区允许删除当前工单已有图片，并允许新增现场图片。

**Given** 用户保存编辑  
**When** 图片上传或 URL 写入失败  
**Then** 表单保留输入并展示失败原因，不应部分更新工单问题信息。

阶段 3 图片能力只服务工单详情编辑态，不处理助手对话图片、图片理解或建单图片保留。

## 刷新与返回契约

- 操作成功后，详情必须展示最新状态、责任人、处理信息和处理记录。
- 从列表返回时，普通列表必须继续保留 `ticketStatus` 和 `ticketSort` 上下文。
- 解决或拒绝后，如果当前筛选仍是待处理，列表中不再展示该工单。
- 指派后，原施工方当前列表不再展示该工单；管理员列表仍展示该工单。
- 重新打开后，责任人的待处理列表重新展示该工单。

## 异常状态契约

### 无权限

**Given** 当前身份无权操作目标工单  
**When** 用户提交任一操作  
**Then** 系统拒绝提交并展示无权限或状态已变化提示，不泄露其他项目数据。

### 工单不存在

**Given** 目标工单 id 不存在  
**When** 用户提交任一操作  
**Then** 系统提示工单不存在或已不可访问。

### 状态过期

**Given** 用户打开详情后，工单状态或责任人已被其他操作改变  
**When** 用户提交旧页面上的操作  
**Then** 系统拒绝提交并提示刷新或返回详情查看最新状态。

### 提交失败

**Given** 上传、更新或处理记录写入失败  
**When** 用户提交操作  
**Then** 系统保留当前输入内容并展示可理解的失败原因。

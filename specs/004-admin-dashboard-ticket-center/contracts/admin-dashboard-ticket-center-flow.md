# UI 契约: 管理员态势大盘与工单中心

## 目标

固定阶段 5 的页面级交互、URL 查询参数、数据口径和验收接口，供 `/speckit-tasks` 和实现阶段拆分任务使用。

## 视图入口

| 视图 | URL | 允许角色 | 说明 |
|------|-----|----------|------|
| 数据大盘 | `/?view=dashboard` | 管理员 | 展示当前项目态势指标、状态分布、严重程度分布和重点关注入口 |
| 管理员工单中心 | `/?view=admin-tickets` | 管理员 | 展示当前项目全部工单、筛选区、表格和详情入口 |
| 工单详情 | `/?view=admin-tickets&ticketId=<id>` | 管理员 | 复用通用工单详情和阶段 3 操作面板 |

非管理员访问以上视图时，必须展示无权限反馈。

## 数据大盘契约

### 必须展示

- 当前项目名称、城市、客户和项目类型。
- 工单总数。
- 待处理、已完成、已拒绝数量。
- 紧急工单数量。
- 状态概览。
- 严重程度分布。
- 紧急工单或重点关注区域。

### 指标下钻

| 点击来源 | 目标 URL 行为 | 预期筛选 |
|----------|---------------|----------|
| 待处理指标 | 进入 `view=admin-tickets` | `adminStatus=pending` |
| 已完成指标 | 进入 `view=admin-tickets` | `adminStatus=completed` |
| 已拒绝指标 | 进入 `view=admin-tickets` | `adminStatus=rejected` |
| 紧急指标或重点关注区域 | 进入 `view=admin-tickets` | `adminSeverity=urgent` |
| 工单总数 | 进入 `view=admin-tickets` | 无额外筛选，展示全部 |

进入工单中心后，筛选区必须让管理员看见自动生效的筛选条件。

## 管理员工单中心筛选契约

### 查询参数

| 参数 | 允许值 | 默认值 |
|------|--------|--------|
| `adminStatus` | `all`、`pending`、`completed`、`rejected` | `all` |
| `adminSeverity` | `all`、`minor`、`normal`、`serious`、`urgent` | `all` |
| `adminSpecialty` | `all`、`architecture`、`structure`、`plumbing` | `all` |
| `adminKeyword` | 任意短文本 | 空 |
| `adminTicketNumber` | 任意短文本 | 空 |

非法枚举值必须回退为默认值。空文本不得作为有效筛选条件。

### 筛选行为

- 多个枚举筛选同时生效时，结果取交集。
- 工单编号搜索优先匹配工单编号。
- 普通关键词匹配问题描述、详细位置和问题详情。
- 清空筛选后回到 `/?view=admin-tickets` 或等效的全部范围。
- 当前筛选无结果时，表格区域展示空结果，并保留筛选区和清空入口。

## 表格契约

### 必须展示列

- 工单编号
- 当前状态
- 严重程度
- 专业类型
- 问题描述摘要
- 创建时间
- 当前责任人
- 发起人
- 查看详情入口

### 展示规则

- 紧急工单必须在视觉上突出。
- 问题描述过长时可以截断，但详情中必须可查看完整内容。
- 表格默认按紧急优先、最新优先展示。
- 查看详情入口必须保留当前管理员筛选上下文。

## 详情联动契约

### 打开详情

- 管理员点击表格行或查看详情入口后，进入 `view=admin-tickets&ticketId=<id>`。
- 若工单存在且属于当前项目，展示通用工单详情。
- 若工单不存在，展示未找到反馈。
- 若工单存在但不属于当前项目，展示无权限反馈。

### 操作同步

- 管理员可按阶段 3 规则执行编辑、解决、拒绝、指派他人和重新打开。
- 操作成功后必须新增处理记录。
- 操作成功后，详情、表格和大盘统计必须在一次刷新、返回或页面更新后展示一致结果。

## 空状态和失败状态

- 当前项目没有任何工单：大盘显示所有指标为 0；工单中心显示当前项目暂无工单。
- 筛选无结果：工单中心显示当前筛选无匹配工单，并提供清空筛选入口。
- 数据读取失败：展示工单读取失败或统计读取失败说明。
- 操作提交失败：保留阶段 3 表单输入并展示失败原因。
